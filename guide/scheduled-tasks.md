# 定时任务

AeroLoongClaw 支持通过自然语言创建定时任务。任务以完整 Agent 身份运行，拥有全部工具权限，可访问群组历史、内存和技能系统。

## 通过自然语言创建任务

在任意群组对话中，直接用自然语言描述你要定时执行的任务即可。例如：

> "每天早上 9 点给我发一份今日天气摘要"
>
> "每隔 30 分钟检查一下服务器状态，有异常就告诉我"
>
> "这周六下午 3 点提醒我开会"

Agent 会调用 `schedule_task` 工具创建任务，无需记忆工具签名。任务创建后，Agent 会发送确认消息，包含任务 ID 和下次执行时间。

::: tip
任务以 Agent 身份运行，这意味着它可以调用 `send_message` 主动推送消息，也可以访问所有已安装的技能和 MCP 工具。
:::

## 调度类型

### Cron 表达式

按标准 cron 表达式调度，支持本地时区（由 `TZ` 环境变量配置）。

| 表达式 | 含义 | 示例输出 |
|--------|------|----------|
| `*/5 * * * *` | 每 5 分钟 | next: 2026-03-26T10:05:00+08:00 |
| `0 * * * *` | 每小时整点 | next: 2026-03-26T10:00:00+08:00 |
| `0 9 * * *` | 每天 9:00 | next: 2026-03-27T09:00:00+08:00 |
| `0 9 * * 1` | 每周一 9:00 | next: 2026-03-30T09:00:00+08:00 |
| `30 8 * * 1-5` | 工作日 8:30 | next: 2026-03-27T08:30:00+08:00 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 | next: 2026-03-26T09:00:00+08:00 |
| `*/15 9-17 * * 1-5` | 工作日 9:00-17:00 每 15 分钟 | next: 2026-03-26T09:15:00+08:00 |

::: warning
Cron 表达式使用本地时区，不是 UTC。所有时间都会转换为本地时区后执行。
:::

### 固定间隔（Interval）

以毫秒为单位的固定间隔，适合需要频繁执行的监控任务。

| 毫秒 | 含义 |
|------|------|
| `300000` | 每 5 分钟 |
| `600000` | 每 10 分钟 |
| `1800000` | 每 30 分钟 |
| `3600000` | 每小时 |
| `86400000` | 每天 |

### 一次性任务（Once）

指定一个本地时间戳，只执行一次。时间格式：`YYYY-MM-DDTHH:mm:ss`（**不要加 Z 后缀或时区偏移量**）。

错误示例：`2026-03-27T15:30:00Z`（带 Z 后缀会报错）
正确示例：`2026-03-27T15:30:00`

## 上下文模式

创建任务时可以指定任务在哪种上下文中运行：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `group` | 在群组当前会话中运行，可访问聊天历史和 Memory | 需要回顾之前讨论、跟进待办事项 |
| `isolated` | 在全新的隔离会话中运行，无聊天历史 | 独立的报告生成、定期检查 |

不确定选哪个时，直接问 Agent："这个任务需要回顾我们的对话吗？"

## 任务管理工具

### `schedule_task` — 创建定时任务

```typescript
schedule_task({
  prompt: string,           // 任务描述（必填）
  schedule_type: "cron" | "interval" | "once",  // 调度类型（必填）
  schedule_value: string,    // 调度值（必填）
  context_mode?: "group" | "isolated",  // 上下文模式，默认 "group"
  target_group_jid?: string, // 跨群调度（仅主群组可用）
})
```

### `list_tasks` — 列出所有任务

```typescript
list_tasks(): {
  // 返回当前群组可见的所有任务列表
  // 格式: - [taskId] prompt... (type: value) - status, next: ISO时间
}
```

### `pause_task` — 暂停任务

```typescript
pause_task({
  task_id: string,  // 要暂停的任务 ID
})
```

暂停后任务不会运行，直到被 `resume_task` 恢复。

### `resume_task` — 恢复任务

```typescript
resume_task({
  task_id: string,  // 要恢复的任务 ID
})
```

### `cancel_task` — 取消任务

```typescript
cancel_task({
  task_id: string,  // 要取消的任务 ID
})
```

::: danger
取消任务不可恢复，任务会被永久删除。
:::

## 任务状态

| 状态 | 含义 |
|------|------|
| `active` | 任务正常运行，等待下次执行 |
| `paused` | 任务暂停，不会被执行 |
| `completed` | 一次性任务已执行完毕 |

## 跨群组调度

只有**主群组（main channel）**的 Agent 可以在其他群组创建定时任务。普通群组的 Agent 只能为自己创建任务。

跨群组调度的典型场景：

- 主群组 Agent 收到指令："每天早上给 family 群发天气"
- Agent 使用 `target_group_jid` 参数指定目标群组
- 任务会在目标群组中运行，并将结果发送到目标群组

非主群组尝试跨群调度时，任务会被拒绝，Agent 会收到警告日志。

## 任务队列与聊天队列的关系

定时任务与聊天消息共享同一个 Group Queue。这意味着：

- **任务和聊天消息统一排队**：如果群组的聊天队列中有待处理的消息，新触发的定时任务会排在它们后面
- **串行执行**：默认情况下，同一群组的任务和聊天消息串行执行，不会并发冲突
- **并行池（isolated 模式）**：`isolated` 模式的任务有独立的并行池，不受 `PER_GROUP_MAX_PARALLEL_CHAT_RUNS` 限制，可以与聊天消息并行运行
- **并发控制**：`MAX_CONCURRENT_CONTAINERS` 控制全局最大容器数，`PER_GROUP_MAX_PARALLEL_TASKS` 控制每个群组的最大并行任务数

::: info
`group` 模式的定时任务使用聊天队列，和普通消息一样需要排队等待。`isolated` 模式的定时任务使用独立并行池，不占用聊天队列位置。
:::

任务运行时，Agent 的输出会实时推送到群组聊天。结果会记录到数据库，包括执行时间、持续时长和状态摘要。
